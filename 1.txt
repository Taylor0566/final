目要求
校园活动管理平台设计。基于基础校园活动信息平台的静态网页，进一步完善功能
并提供更强的交互性，更好地实现校园内各类活动的信息发布与参与管理，方便学生获
取活动资讯并积极参与，从而构建一个结构完善、布局合理、内容丰富、交互性强的校
园活动管理平台。具体要求如下：
（1）设计要点：
网站主体框架包括：活动发布与展示、活动报名与组织、活动日历与提醒、社交互动与评价、统计分
析以及个性化推荐等模块。
1. 用户登录与个人中心：
o 用户登录：实现用户注册、登录与注销功能，确保账户安全。
o 个人中心：创建用户个人中心页面，用户可以查看已报名的活动、编辑个人资料等。
o 收藏与历史记录：提供用户的收藏功能（收藏感兴趣的活动）和历史记录功能（查看
已参加的活动），方便管理个人活动信息。
2. 搜索与推荐功能：
o 活动搜索：提供高效的活动搜索功能，支持根据关键词或类别查找活动，并在输入时
实时提示相关结果。
o 个性化推荐：根据用户兴趣偏好或参与记录推荐相关校园活动，例如用户常参加的活
动类型或当前热门的活动。
3. 活动详情页：
o 多维展示：提供活动的详细信息页面，包括活动时间、地点、简介等内容，并配以活
动照片或宣传视频等多媒体资料，让用户全面了解活动信息。
o 用户评价：添加用户评论模块，展示并允许用户发表对活动的评价与建议，营造良好
的互动交流氛围。
o 提醒功能：显示活动开始前的倒计时信息，并支持用户设置提醒，以便及时收到活动
通知。
4. 活动发布与用户分享：
o 发布页面：提供活动发布页面，允许管理员或活动组织者录入并发布校园活动信息，
包括活动标题、时间、地点、简介和海报等内容。
o 分享功能：支持用户将感兴趣的活动通过链接或二维码分享给他人或发布到社交媒体
平台，扩大活动影响力。
5. 活动报名与互动：
o 报名功能：提供在线报名功能，用户可在活动详情页点击报名参加活动，并在个人中
心查看自己的报名状态。
o 互动增强：允许用户点赞感兴趣的活动或收藏活动，增加用户互动性和参与度。
6. 评论与反馈：
o 评论功能：用户可针对校园活动发表评论，分享观点或提出问题。
o 意见反馈：设置用户反馈通道，如反馈表单或联系邮箱，用于收集用户对平台功能和
体验的意见与建议，以便平台后续改进。
7. 数据可视化：
o 统计图表：利用 Echart 等工具对平台数据进行可视化展示，例如不同类别活动的数量
分布、各活动的参与人数统计、活动受欢迎程度排行等。
o 数据洞察：通过直观的图表展示上述数据，帮助用户和管理者了解校园活动的整体情
况和趋势。
8. 移动端适配：
o 响应式设计：采用 Bootstrap 等前端框架实现响应式布局，使网站能够适应 PC 端和移
动端的不同屏幕尺寸。
o 移动优化：针对手机等移动设备优化导航菜单、内容布局和交互方式，确保用户在移
动端获得良好的浏览与操作体验。
以上设计要点主要提供思路，可根据实际情况选择 2-3 个要点完成即可。
（2）技术要求：
综合运用 HTML / CSS / JavaScript / Echart / jQuery / Bootstrap 等前端技术，
共同实现一个功能完善、布局合理、内容丰富、用户友好的综合网站，具体包括：
HTML 网页的基本结构构建，HTML 基本元素（文本、段落和列表）的合理运用，
有关超链接和多媒体文件（图片、动画、声音）的应用；对网站页面进行合理排版布局 ，
使用 CSS 样式优化页面显示效果；基于 JavaScript 实现页面的动态交互和逻辑处理
（如：动态轮播特效, 表单提交, 点击事件等等）；适当采用 Echart 用于展示图表数据，
提升页面的可视化效果；合理运用 Bootstrap 提供响应式设计和组件库等等。

1． 结构布局(10 分)：结构清楚合理，页面间有较强连贯性，便与浏览查找；
2． 视觉效果(10 分)：网站整体风格一致，图片的制作美观实用，页面美观；
3． 交互性(20 分)：网站具有较好的交互性，提供良好的用户体验；
4． 创意(20 分)：页面具有个人设计风格，具有观赏性，内容有创意；
4. 技术应用(40 分)：尽可能使用更多技术要点，包括但不限于 HTML / CSS /
JavaScript / Echart / jQuery / Bootstrap 等等。
